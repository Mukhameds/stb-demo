# STB-DEMO

Минимальный детерминированный прототип сигнально-реактивной адаптивной архитектуры.

---

## Что представляет собой этот репозиторий

STB-DEMO — это компактный инженерный прототип, демонстрирующий:

* онлайн-формирование структур из повторяющихся сигналов,
* генерацию предсказаний на основе структуры,
* локальную адаптацию, управляемую ошибкой,
* ограниченный рост с контролируемым удалением,
* детерминированное и полностью наблюдаемое поведение.

Этот репозиторий не является AI-моделью, оптимизированной под конкретную задачу.
Это проверка архитектурного подхода на уровне системы.

---

## Модель выполнения

Каждый тик обрабатывает сигналы через локальные реакции:

```
Signal → Block.React() → new Signals → competition → winner → next tick
```

В системе нет:

* глобального контроллера,
* централизованной функции политики,
* отдельной фазы обучения.

Всё поведение возникает из распространения сигналов между независимыми блоками.

---

## Основные понятия

### Сигнал

Дискретное событие, содержащее:

* тип (вход, структура, предсказание, ошибка, действие),
* значение,
* массу активации.

Сигналы — единственный способ взаимодействия компонентов.

---

### Блок

Автономная вычислительная единица, которая:

* реагирует на входящие сигналы,
* может испускать новые сигналы,
* хранит только локальное состояние.

Блоки не координируются напрямую.
Они влияют друг на друга только через общее сигнальное поле.

---

## Реализованные механизмы

### 1. Формирование структуры

Повторяющиеся входные паттерны накапливают внутреннюю массу.

Когда достигается порог, создаются новые структурные блоки:

* Парные блоки: `[A-B]`
* Последовательностные блоки: `(A>B)`
* Композиционные блоки: `[[A-B]-C]`

Это обеспечивает инкрементальный рост структуры без обучения на датасете.

---

### 2. Структурное предсказание

Структуры поддерживают статистику переходов.

При достаточной активации структура формирует ожидание:

```
[A-B] ⇒ C
```

Предсказание возникает из доминирования структуры, а не из отдельного инференс-прохода.

---

### 3. Адаптация, управляемая ошибкой

Если фактический вход противоречит активированному ожиданию:

* генерируется сигнал ошибки,
* веса неверного перехода уменьшаются,
* альтернативные переходы усиливаются,
* временное усиление ускоряет адаптацию.

Никакого цикла переобучения не выполняется.
Адаптация происходит онлайн.

---

### 4. Конкуренция и ингибирование

Активированные структуры накапливают массу.

Самая сильная структура:

* становится победителем текущего тика,
* подавляет альтернативы через ингибирование.

Это стабилизирует поведение без централизованной логики выбора.

---

### 5. Энергетическое ограничение

Действия потребляют энергию.

Низкий уровень энергии уменьшает силу сигнала.

Это вводит простое ресурсное ограничение в динамику активации.

---

### 6. Контролируемое забывание

Неактивные обученные блоки удаляются после заданного окна времени.

Структуры сохраняются, если они:

* остаются предиктивными,
* содержат значимую массу переходов,
* были недавно активны.

Это предотвращает неограниченный рост структуры.

## Демонстрационный сценарий

Запустите интерактивную демо-программу:

```
go run main.go
```

Затем выполните:

```
demo
```

Демонстрация проходит в три этапа:

### Этап 1 — Формирование структуры

Повторяющиеся последовательности:

```
1 2 1 2 1 2
2 3 2 3 2 3
```

Результат:

* формируются парные структуры `[1-2]` и `[2-3]`.

---

### Этап 2 — Стабильное предсказание

Повторяющийся паттерн:

```
1 2 3
```

Результат:

* структура `[1-2]` стабильно предсказывает `3`,
* уверенность предсказания увеличивается.

---

### Этап 3 — Ошибка предсказания и адаптация

Вход:

```
1 2 4
```

Результат:

* обнаруживается ошибка,
* применяется ингибирование,
* предсказание переключается с `3` на `4` после повторяющихся подтверждений,
* цикл переобучения отсутствует.

Все внутренние события отображаются в логах.

## Что валидирует этот прототип

Этот прототип демонстрирует, что:

* онлайн-структурное обучение возможно без отдельной фазы тренировки,
* предсказание может возникать из конкуренции структур,
* адаптация может происходить через локальные сигналы ошибки,
* структурное удаление может ограничивать рост системы,
* весь процесс может оставаться детерминированным и наблюдаемым.

---
