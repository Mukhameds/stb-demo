# STB-DEMO

**Минимальная реализация сигнально-реактивной адаптивной архитектуры ИИ**

---

## Обзор

STB-DEMO — это рабочий прототип, реализующий базовые механизмы архитектуры СТБ (Сигнальная Теория Бытия).

Демо показывает, как адаптивная система может:

* формировать внутренние структуры из повторяющихся сигналов,
* генерировать предсказания на основе этих структур,
* обнаруживать ошибки предсказания,
* перестраивать себя без переобучения,
* забывать нестабильные гипотезы,
* сохранять детерминированное и трассируемое поведение.

Это не задача-ориентированная модель ИИ.
Это архитектурный валидационный прототип.

---

## Что реализовано в демо

Система построена вокруг минимального реактивного вычислительного цикла:

```
Сигнал → Блок → Реакция → Новый сигнал
```

Где:

* **Сигнал** — дискретное внешнее или внутреннее событие.
* **Блок** — автономная структурная единица, реагирующая на сигналы.
* **Реакция** — локальное обновление состояния, порождающее новые сигналы.

Глобальное поведение возникает из каскадов локальных реакций.

---

## Основные реализованные механизмы

### 1. Накопление сигналов и формирование структур

Повторяющиеся входные последовательности (например, `A B C`) постепенно накапливают “массу”:

* PAIR-блоки: `[A-B]`
* SEQ-блоки: `(A>B)`
* COMPOSE-блоки: `[[A-B]-C]`

Когда масса превышает порог, создаётся новый структурный блок.

Это заменяет обучение на датасетах постепенным структурным ростом.

---

### 2. Предсказание без инференса

Предсказание не вычисляется через модель.

Вместо этого:

* стабильные структуры формируют предиктивные веса,
* активируется наиболее сильная гипотеза,
* уверенность в предсказании растёт при подтверждении.

Пример:

```
[A-B] ⇒ C
```

Предсказание возникает из структуры, а не из прямого прохода по нейросети.

---

### 3. Обнаружение ошибки

Если фактический вход противоречит ожидаемому:

```
ERROR = [[A-B]: C → Z]
```

Система:

* снижает уверенность неверной гипотезы,
* активирует ингибирование,
* увеличивает коэффициент обучения (error-boost).

Переобучения не происходит.

---

### 4. Структурная адаптация

После повторяющихся противоречий:

* формируются новые структурные блоки,
* прежнее доминирующее предсказание деградирует,
* система детерминированно переключается.

Пример:

```
[A-B] ⇒ C   (сначала)
[A-B] ⇒ Z   (после серии ошибок)
```

Адаптация происходит через перестройку структуры, а не через backpropagation.

---

### 5. Ингибирование и забывание

Конкурирующие гипотезы:

* подавляются локально,
* постепенно деградируют,
* удаляются при отсутствии использования.

Это предотвращает неконтролируемый рост графа и сохраняет стабильность.

---

### 6. Детерминированность и трассируемость

Каждое внутреннее событие отображается в логах:

* создание блоков,
* обновление предсказаний,
* фиксация ошибки,
* активация ингибирования,
* динамика энергии.

Система полностью наблюдаема и проверяема.

---

## Что демонстрирует этот прототип

Демо подтверждает, что:

* адаптивное поведение может возникать из локальных реакций сигналов,
* предсказание может формироваться из структуры, а не из инференса,
* ошибки могут перестраивать систему без переобучения,
* структурная память может заменить обучение на датасетах,
* забывание может быть естественным и постепенным.

Это подтверждает, что интеллект можно реализовать как непрерывную адаптивную систему, а не как статическую модель.

---

## Архитектурное отличие от нейросетей

Нейросети:

* обучаются офлайн,
* выполняют инференс на каждый запрос,
* требуют переобучения для структурной адаптации,
* функционируют как модель-сервис.

STB-DEMO демонстрирует альтернативу:

* непрерывную эволюцию структуры вместо циклов обучения,
* локальные реакции вместо глобального инференса,
* структурную память вместо зависимости от датасетов,
* детерминированное и трассируемое поведение.

Цель демо — не превзойти нейросети по задачам,
а продемонстрировать иной вычислительный фундамент.

---

## Назначение репозитория

Это не продакшн-система.

Это минимальный инженерный прототип, подтверждающий:

* сигнально-реактивные вычисления,
* структурное обучение,
* автономную адаптацию,
* экономическую масштабируемость событийных ИИ-архитектур.

Цель — заложить новую системную основу для адаптивного ИИ.
